# Avito Frontend Trainee Assignment — Spring 2026

Веб-приложение — личный кабинет продавца с AI-ассистентом для улучшения объявлений.

## Стек

- React 19 + TypeScript
- Vite + Tailwind CSS v4
- shadcn/ui — UI-компоненты
- TanStack Query — запросы к API
- Zustand — стейт-менеджмент
- react-hook-form + Zod — формы и валидация
- Vitest + Testing Library — тесты

## Запуск

### 1. Сервер
```bash
cd server
npm install
npm start
# сервер запустится на http://localhost:8080
```

### 2. Настройка LLM

**Вариант A — Ollama (локально, бесплатно):**
```bash
# Установить Ollama: https://ollama.com
ollama pull llama3
ollama serve
```

**Вариант B — Внешний API (Grok / OpenAI):**

Получи ключ на [x.ai](https://x.ai) или у другого провайдера.

### 3. Переменные окружения

Скопируй `.env.example` в `.env` и заполни:
```bash
cp .env.example .env
```
```
VITE_API_URL=http://localhost:8080

# Ollama
VITE_LLM_API_URL=http://localhost:11434
VITE_LLM_MODEL=llama3

# Внешний API (раскомментировать если не Ollama)
# VITE_LLM_API_KEY=your_key_here
# VITE_LLM_API_URL=https://api.x.ai
# VITE_LLM_MODEL=grok-3-mini
```

### 4. Клиент
```bash
npm install
npm run dev
# клиент запустится на http://localhost:5173
```

## Запуск через Docker Compose

Требует установленного [Docker Desktop](https://docs.docker.com/get-started/get-docker/).
```bash
# В корне репозитория
docker compose up --build
# клиент → http://localhost
# сервер → http://localhost:8080
```

Для внешнего LLM API создай `.env` рядом с `docker-compose.yml`:
```
VITE_LLM_API_KEY=your_key_here
```

## Функциональность

### Обязательное
- Страница списка объявлений `/ads` — поиск, фильтры, сортировка, пагинация
- Страница просмотра `/ads/:id` — полная информация, блок незаполненных полей
- Страница редактирования `/ads/:id/edit` — форма с динамическими полями по категории
- AI-ассистент: «Улучшить описание» и «Узнать рыночную цену»
- Сохранение черновика в `localStorage`
- Отображение состояний загрузки (скелетоны) и ошибок

### Дополнительное ★
- Переключатель grid/list на странице списка
- Тёмная тема с сохранением в `localStorage`
- AI-чат с контекстом объявления на странице редактирования
- Визуальное сравнение «Было → Стало» при улучшении описания
- Отмена запросов при переходе между страницами (AbortSignal)
- Docker Compose для запуска клиента и сервера
- Юнит-тесты (Vitest + Testing Library)

## Принятые решения

**Категория не редактируется** — API принимает категорию в PUT-запросе, но сервер
игнорирует её изменение (берёт из оригинального объявления). Чтобы не вводить
пользователя в заблуждение, поле категории показывается как disabled.

**Черновик и серверные данные** — при открытии формы редактирования сначала
проверяется наличие черновика в `localStorage`. Если черновик есть — загружается он,
иначе данные с сервера. После сохранения или отмены черновик очищается.

**Diff «Было → Стало»** — реализован word-level diff без внешних библиотек.
Удалённые слова подсвечиваются красным зачёркиванием, добавленные — зелёным.
Показывается только если у объявления уже было описание.

**LLM-интеграция** — поддерживаются оба варианта: Ollama (локально) и
OpenAI-совместимый API. Переключение происходит автоматически по наличию
переменной `VITE_LLM_API_KEY` в окружении.

**Реализация сортировки по цене на backend** — добавлен вариант сортировки по цене на серверной стороне.

**Нет идеального следования макету, т.к. данное требование не указано в ТЗ.**
